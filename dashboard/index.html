💡 What is a Quality Gate?
1

ZAP Scans Your Website

GitHub Actions automatically starts OWASP ZAP when you push code. ZAP visits your website and tests for security vulnerabilities.

2

Results are Counted by Severity

Each vulnerability is categorized as High, Medium, Low, or Informational.

3

Counts are Compared to Thresholds

The quality-gate.py script checks: "Did we find more than the allowed number?" Currently: High ≤ 0, Medium ≤ 5, Low ≤ 20.

4

Build PASSES or FAILS

If any threshold is exceeded → the pipeline exits with code 1 → GitHub marks the build as ❌ FAILED. Your code will NOT be deployed until you fix the issues.

⚙️ How to Change Thresholds

Open .github/workflows/zap-dast-scan.yml and change these values:

env:
  MAX_HIGH_ALERTS:    0   # Change to 2 to allow up to 2 HIGH findings
  MAX_MEDIUM_ALERTS:  5   # Change to 10 to allow up to 10 MEDIUM findings
  MAX_LOW_ALERTS:    20   # Change to 50 to allow up to 50 LOW findings
📖 Understanding Severity Levels

🔴 High Severity

Very dangerous. An attacker can exploit this to steal data, take over accounts, or damage the system.

Action: Fix IMMEDIATELY before merging.

🟡 Medium Severity

Significant issue that could be exploited given the right conditions.

Action: Fix within this sprint or current release cycle.

🔵 Low Severity

Minor issue — harder to exploit but still worth fixing.

Action: Track and fix in upcoming sprints.

⚪ Informational

Not a vulnerability — just useful info about your app's configuration.

Action: Review and decide if action is needed.

🔧 Step-by-Step Fix Guide
1

Download the HTML Report

Go to GitHub → Actions → your latest workflow run → scroll to Artifacts → download security-reports-run-N.zip. Open full-report.html in your browser.

2

Find the HIGH Severity Findings

In the HTML report, look for red-highlighted alerts. Each alert shows the affected URL, the attack payload, and a Solution section explaining how to fix it.

3

Use OWASP Resources to Learn the Fix

Every alert has a CWE ID (e.g., CWE-89 = SQL Injection). Search OWASP Top 10 or OWASP Cheat Sheets for detailed fix guides.

4

Is it a False Positive? Suppress It

If ZAP flagged something that isn't actually a vulnerability, add its rule ID to zap/rules.tsv with action IGNORE. Include a comment explaining why.

5

Push Your Fix and Re-run the Pipeline

Once you fix the issues, push to GitHub. The pipeline will automatically re-run the ZAP scan. If all thresholds are met, the build will ✅ PASS.
